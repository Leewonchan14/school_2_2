자바 버츄얼 머신(java virtual machine)에 의해 네트워크에 연결된 어느 클라이언트에서나 실행됨(자바 프로그램의 구동프로그램, .class파일 실행함)

바이트 코드 : 가상머신에서 구동가능한 바이너리 코드(.class파일)


c / c++ : 모든 소스코드를 모아 exe파일로 만들어 실행(메모리가 적은경우 낭패)
소스코드를 오브젝트 파일로 만들어 링커가 최종적으로 실행파일로 만든다.
java : 여러 소스코드가 있긴하지만 모아서 실행파일로 만드는 과정 없음, 한번 작성하면 수정하지않고 어디에서나 실행할수있다(WORA)
.java 파일을 .class파일로 바꾼다음 .bc(바이트코드) 파일로 최종변환한다. .bc코드는 가상머신에서 사용가능하다.

자바언어의 특징:
1. 독립성 : 어디서나 구동가능한 언어이다.
2. 객체지향 언어이다. 인산의 사고에 가깝게 구현가능
3. 캡슐화 : 클래스 밖에 선언된 변수, 메소드는 존재 하지 않는다.
4. 가비지 컬렉션 : 할당된 메모리 반환을 자동으로 해준다.(언제 실행될지 모르기 때문에 실시간 응용프로그램에는 부적절)


개발 주의사항
1. 클래스이름과 파일이름은 같아야한ㄴ다.(퍼블릭 클래스는 하나여만 하며 퍼블릭 클래스와 파일이름이 같아야 한다.)

2.main함수를 쓰기위해 앞에 public static void main(String[] arg)를 작성한다.
