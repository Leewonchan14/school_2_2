명령어 피연사자의 데이터 주소 지정방식 4가지

1. 인네데이터 어드레싱 : 명령어 안에 데이터값 저장

2.레지스터 어드레싱 : 레지스터 안에 데이터 저장후 레지스터 인덱스를 명령어에 표현

3. 베이스 어드레싱 : 메모리 안 데이터 저장후 명령어에 포함되어있는 레지스터에 메모리 주소가 저장됌

4. pc리얼티브 어드레싱 : 메모리 안에 다음명령어의 주소가 들어있고 명령어 안에 

프로그램 카운터(pc)<= 다음에 실행할 명령어의 주소가 들어있다.

스택 프레임 : 스택 차곡 차곡 쌓인다.



명령어 종류

1. 논리, 산술 연산 명령어

2. 데이터를 읽거나 쓰는 명령어

3. 함수 호출, 조건문, 반복문 명령어=>컨디셔널 점프, 언컨디셔널 점프(끝으로 가는 명령어)


ㄱ. 산술, 논리 명령어

피연산자 3개
add a b c => a=b+c
sub a b c => a=b-c
자주 쓰는 피연산자는 레지스터에 산술 연산 하면 빠르다.

레지스터 vs 메모리
메모리에서 레지스터로 데이터를 가져와 계산한다.

작은 상수의 데이터는 명령어 안에 저장한다.

ㄴ.R-Format 명령어들(레지스터들만 사용)
ㄷ.I-Format 명령어들((상수값을 갖는 ,레지스터에 메모리 데이터 저장하는 연산에만)
ㄹ.S-Format 명령어들

2. 데이터를 읽거나 쓰는 명령어
