1. 메세지 인증(authentication)
	: 올바른 송신자에게 온 것이다 라는 성질

ex) 앨리스가 A은행에게 메세지를 보낸다 자기 계좌에서 밥 계좌로 돈을 보내라

A은행이 해야할일
	1. 출처 인증(앨리스가 쓴 메세지가 맞는지)
	2. (통신중)내용 변경 됬는지 아닌지

2. 메세지 인증 코드(MAC)

	: 무결성 확인, 메세지에 대한 인증을 위한 코드(
	
	1. 입력 : 메세지, 공유하는 키
	2. 출력 : 고정 비트 길이의 코드

3. 메세지 인증 코드 이용 순서
	
	1. 사전에 앨리스와 은행은 키를 공유
	2. 앨리스는 메세지와 키를이용한 MAC값을 같이 보냄
	3. 은행은 메세지를 사전에 키를이용해 MAC값을 구한후 수신받은 MAC값이랑 비교	
	4. 같아야 인증

4. 키배송 문제
	1. 공개키 암호
	2. Diffie-Hellman 키교환
	3. 키배포 센터
	4. 안전한 방법 으로 보내기

5. 메세지 인증 코드 ex)
	1. 스위프트
		1) 은행과 은행 사이
		2) 무결성 확인, 메세지 인증코드 사용

	2. IPsec 
		1) IP기능 추가 한것
		2) 인증, 무결성위해 메세지 인증코드 이용
	
	3. SSL/TLS
		1) 웹,온라이 쇼핑등에서 사용
		2) 인증,무결성 위해 메세지 인증코드 이용

6.인증 코드의 실현 방법
	1. HMAC
		: 해시함수를 이용한 실현
	2. AES
		: 같은 블럭암호를 이용한 실현

7. 인증 암호
	1. Encrypt-then-MAC
		: 평문을 암호화 후 MAC값을 계산
	2. Encrypt-and-MAC
		: 평문 암호화 이후 그와 별도의 평문으로 MAC값을 얻는것
	3. MAC-then-Encrypt
 		: 미리 MAC값을 얻고 평문과 같이 암호화

8. GCM(Galois/COunter Mode)
	: 인증 모드의 일종
	: 128비트 블럭 암호를 CTR모드를 이용해 MAC값을 얻는다. (병렬가능, 키관리 편함)

8. 일방향 해시함수를 이용한 메세지 인증코드
	그림보고 정리!

	opad = opad@key

9. 당할수 있는 공격
	1. 재전송공격
		멜로리가 A은행에게 "100만원을 계좌에 넣으라"고 메세지를 보낸다
		A은행은 B은행에게 메세지와 MAC값을 B은행에게 넘긴다.
		B은행은 계좌에 100만원을 넣는다.	
		A>B로가는 메세지와 MAC값을 도청, 보존 한뒤 B은행에게 다이렉트로 메세지를 보낸다.(여러번 반복)
	막는방법
		1. 순서 번호 : 진짜 송신자에게 온 메세지마다 1회씩 증가시키는 번호 매김
		2. 타임 스탬프 : 보낸 시간 저장
		3. 비표 : 송신자에게 랜덤한 비표값을 전송, 비표와 함계 MAC값을 계산

	2. 키 추측 공격
		인증코드에 대한 공격(전사공격,생일공격)
