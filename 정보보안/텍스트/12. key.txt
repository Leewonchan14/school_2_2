||1. 암호기술에서 사용되는 키||
	1. KEY
		1. key는 굉장히 큰수
		2. 키 공간이 중요
		3. 키 공간은 비트길이로 결정
		4. 평문과 동일한 가치
		
	2. 각 암호기술의 키 공간
		
		1. DES : 56비트
		2. RSA : 1024비트

	3. key의 종류
		1) 기밀성을 위한 키
			대칭암호,공개키 암호에서 사용하는 키
		2) 인증을 위한 키
			메세지인증, 디지털 서명에 사용하는 키
		3) 세션키
			통신때 한번만 사용되는 키
		4) 마스터 키
			반복적으로 사용되는 키

		5) 정보 암호화 키 : CEK

		6) 키를 암호화 하는 키 : KEK


||2. 키관리||
	
	1. 키 생성
		1. 난수로 만든다.(의사 난수) : 반드시 암호용 설계 생성기 사용한다.(예측불가능 성질이용)

		2. 패스워드 : 그대로 사용하지 않고(salt라는 난수를 추가해) 해시값으로 사용(사전공격 방어)

	2. 키 배송
		1. 사전공유,키배포 센터, 공개키 암호

	3. 키 갱신
		송신자와 수신자가 동시에 같은 방법으로 정기적으로 키를 교환한다.
		: 과거 통신기록을 방어할수있다. ( 백워드 시큐리티  라고 한다.)

	4. 키 저장
		키보존 문제의 대두!!
		키를 암호화하는 키 ( KEK )
		: 수많은 키중 KEK만 보호하면 모든키를 지킬수있다.
	
	5. 키 폐기
		불필요한 키는 확실히 삭제!!

||3. DIFFIE-HELLMAN 키교환|| => 중간자 공격 가능!!!
	1. 키 교환
		: 공개되어도 상관없는 정보를 두사람이 교환 하면 공통의 비밀값을 얻는다.
	2. 절차
		1) P(소수)와 G(원시근)을 공개되어도 상관없으니 공개한다.
		2) 앨리스와 밥은 G^A mod P, G^B mod P 의 값을 서로 교환(공개 가능)
		3) 앨리스는 A를 만들고 G^B mod P를 수신받았으니 (G^A)^B mod P = G^AxB mod P값을 알수있다. (키 교환 완료)
		4) 밥은 B를 만들고 G^A mod P를 수신받았으니 (G^B)^A mod P = G^AxB mod P값을 알수있다. (키 교환 완료)

	3. 원시근
		: 나머지로 나올수있는 모든 값을 다 가질수있다.

	4. 구체적 원시근 예
		A = 9,	B = 7,	P = 13,	G = 2 일때 교환되는 키값 : 8


||4. 패스워드를 기초로한 암호= PBE||

1. 절차
	1) CEK 로 메세지 암호화
	2) KEK = 해시함수(패스워드 + 솔트(랜덤값))
	3) KEK로 CEK 암호화
	4) 솔트와 암호화 된 메세지를 같이 보관 
	5) 패스워드만 머리속으로 암기
	6) 솔트의 역할 = 사전공격 방어!!

2. 스트레칭
	: 일방향 해시함수 적용 횟수를 증가하는 방법!!

||5. 안전한 패스워드를 만들려면||
	1.  자신만이 알수있는 정보 사용
		1) 중요한 것의 이름 사용금지
		2) 자신에 관한 정보 금지
		3) 타인이 보기쉬운 정보 금지

	2. 복수의 패스워드를 사용
		1) 하나의 패스워드로 여러 용도로 사용하면 안됨

	3. 메모를 유용하게 사용할 것
		1) 물리적인 키와 동일하게 취급 하여 유용하게 사용

	4. 패스워드의 한계
		8문자or숫자의 패스워드는 전사공격이 가능한 수준

	5. 패스워드 생성/관리 툴 이용
		브라우저의 관리 기능이용(신뢰 가능?)
