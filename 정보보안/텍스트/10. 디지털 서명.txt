디지털 서명
	: 앨리스 개인키(서명용 키)를 메세지에 작성( 서명 )
	  밥은 검증용 키(누구나 가질수있다.)를 사용해 앨리스 키를 검증(앨리스것이 맞는지)

||1. 서명 작성과 서명검증||
	1. 서명 작성
		: 메세지의 서명을 작성하는 행위 = 메세지를 개인키로 암호화 한다.
	2. 서명 검증
		: 메세지의 서명을 검증하는 행위 = 공개키로 복호화 하여 검증한다.
	||3. 결론!!!!!||
		: 암호화키와 복호화 키가 나뉘어져있는 공개키암호의 역순버전이다.

||2. 디지털 서명방법 두가지||
	1. 메세지에 직접 서명
		1) 앨리스는 서명된 메세지 따로 암호화된 메세지 따로 밥에게 보낸다
		2) 밥은 서명된 메세지를 공개키로 복호화한 메세지와 복호화된 메세지를 비교하여 같으면 검증된다.
	2. 해시값에 서명 방법
		1) 메세지를 개인키로 직접 암호화 하지않고 해시값을 암호화 하여 밥에게 보낸다.

||3. RSA에 의한 디지털 서명||
	1. 서명 = (메세지)^D mod N
	2. 메세지 = ( 서명 )^E mod N
	3. (메세지, 서명) 을 수신받는다.

||4. 기타 서명방식||
	1. ElGamal 방식
		: mod N 의 이산대수를 구하는 것이 곤란하다는 것을 이용
	2. DSA(Digital Signature Algorithm)
	3. ECDSA
		: 타원 곡선 암호를 사용
	4. Rabin
		: mod N 으로 평방근을 구하는 것이 곤란하다는 것을 이용

||5. 디지털 서명에 대한 공격||
	1. 중간자 공격
		1) 서명 공개키가 정확한 상대의 것인지 확인 필요

	2. 해시 함수에 대한 공격
		1) 함수는 충돌 내성을 가져야함

	3. 공격자의 교묘한 속임수
		1) 사회공학적 공격(생각)

	4. 밥의 어리석은 행동
		의미를 모르는 메세지에는 절대 서명하지 않는다.
	
	5. 잠재적 위조
		RSA에서 랜덤한 비트열 S를 공개키로 암호화한 M을 가진다면 S는 M의 디지털 서명이다.
		
		막는 방법 : RSA - PSS : RSA를 개량한것 ( 해시값을 서명!! 메세지가 아니라 )

	6. 공개키 암호의 공격방법 대부분이 디지털 서명방법에 공격가능
