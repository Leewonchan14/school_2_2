|| 데이터 모델 ||
	= 데이터 구조 + 연산 + 제약조건

1. 관계 데이터 연산
	: 원하는 데이터 얻기위해 필요한 처리 연산을 수행하는것
	1) 관계 대수 
		1. 데이터 처리 과정을 순서대로 기술(절차 언어)
		2. 릴레이션을 처리하는 연산자들의 모임(8가지)

			1)일반 집합 연산자(4가지)
				1.합집합
				2.교집합
				3.차집합
				4.카티션 프로덕트 : 내적같이 연산
				
				일반 집합 연산자의 특징 : 피연산자 2개 필요, 차수가 같아야함, 대응되는 속성의 도메인이 같아야함
				
			2)순수 관계 연산자
				
				1. 셀렉트(시그마) : 조건에 맞는 튜플 반환
					기호..조건(DB) : Select ID,직업 from Custom where ID = 'apple'

				2. 프로젝트(파이) : 주어진 속성들의 값으로만 구성된 튜플 반환
					기호..속성,속성(DB)

				3. 조인
					1) 자연조인(리본) : 공통속성을 이용해 R과S의 연결하여 새로운 튜플 반환( 속성의 합집합)
						R 리본 S ( 갯수가 작은놈으로 맞춘다. )
					2) 동일 조인 : 고객 리본 |고객아이디 = 주문고객| 주문
						: 겹치는것을 하나만 쓰는게 아니라 또 쓴다.
					3) 세미 조인 : R의 차수가 피해입지 않음( R,S의 교집합과 R의 자연조인)
					4) 외부 조인 : 갯수가 큰놈으로 맞춘다.

				4. 디비전(나누기) :   S의 모든 튜플과 관련이 있는 R의 튜플들 반환( 강의자료 그림!!!!)
					R % S (만족하는 튜플들중 S와 중복속성 제거후 반환)


		3. 폐쇄 특성이 존재함
	2) 관계 해석 : 원하는 결과를 얻기 위해 처리원하는 데이터 기술
	3) 위 둘의 역할
		: 유용성을 검증
