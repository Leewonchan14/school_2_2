	데이터 베이스 시스템의 개념

데이터베이스 : 저장소
관리 시스템 : 관리하는 소프트웨어
시스템 : 위 둘을 포함하는 개념



	4... 데이터 베이스 사용자의 분류

1)일반 사용자 = 최종 사용자(USER)
ㄱ.그냥 접근하는사람
ㄴ.지식없이 접근하는사람
ㄷ.지식이 있고 데이터 언어 사용가능, DBMS에 요청

2) 응용 프로그래머
ㄱ.데이터베이스 전문지식으로 응용프로그램을 개발목적으로 접근하는 사람
ㄴ.전문 개발자로 다양한 언어구사, 데이터언어 구사 가능, DBMS명령어 작성능력을 갖춘사람

3) 데이터베이스 관리자
ㄱ.데이터 베이스 구축, 운영, 통제하는 사람
ㄴ. 시스템 관리 총괄, 모든 권한, 책일을 갖는 전문가

	5. 데이터 언어 : 데이터베이스를 조작할수 있는 언어

1) 데이터 정의어
구조 정의, 변경하는 명령어 집합

2)데이터 조작어
입력, 수정, 삭제 및 검색을 위한 명령어들 

3) 데이터 제어어
제어 및 통제 위한 명령어들(각종 제약, 옵션 설정)



	6... 데이터베이스 서버
1)데이터 베이스가 구동되는 서버의 역할을 하는 컴퓨터
2)정보시스템의 규모와 용도에 맞게 적절한 사양의 컴퓨터 선택


	7...데이터 모델
데이터 모델 : 저장구조를 명세하기 위한 개념, 틀

1) 모델의 종류

ㄱ.계층형 데이터 모델
트리와 비슷한 자료 구조

ㄴ. 네트워크 형 데이터 모델
계층형은 1:다 였지만 네트워크는 다:다 이다(그래프 비슷)

ㄷ. 관계형 데이터 모델
데이터 연관관계를 표(테이블)의 키(key)열 을 통해 표현하는 저장방식

ㄹ. 객체 지향형 데이터 모델
테이블들을 트리마냥 이어놓은 구조

ㅁ. 객체 관계형 데이터 모델
관계형 + 객체
	

	8...스키마
	2-1... 스키마(c의 구조체와 비슷)
데이터 구조,제약조건등을 정의한것
ex)학생(학번, 이름, 학년, 성별)

	2-2 스키마의 종류

1) 외부 스키마(사용자 관점)
ㄱ.사용자가 바라보는 관점에서의 구조정의
ㄴ. 일부의 데이터를 사용자마다 다르게 봄(서브 스키마 라고도 부름)
ex) 강의정보(교수), 수강정보(학생)

2) 개념 스키마(컴퓨터 관점)(외부스키마를 다 합친것?)
ㄱ.모든 사용자들의 관점, 전체 조직관점에서 구조 정의한것
ㄴ.전체의 데이터베이스의 논리적 부분 표현
ㄷ. 데이터베이스에는 하나의 개념 스키마만 존재
ㄹ. 보통 스키마 라고 하면 개념스키마
ex) '대학정보' 개념 스키마(대학)

3) 내부 스키마(개발자 관점)

ㄱ. 저장장치의 관점에서 전체 데이터베이스내부구조 정의
ㄴ. 데이터를 어떻게 저장할지에 대한 내부 스키마


	9... 데이터 사전(데이터 베이스의 구조~)(데이터독립성을 실현시키기위해서)
1 전체를 데이터 라고 했을때

데이터 = 사용자 데이터베이스 + 시스템 데이터 베이스
	1) 사용자 데이터 : 실제로 사용자가 이용할수있는 데이터 저장소
	2) 시스템 데이터 베이스 : 데이터 디렉토리 + 데이터 사전
		1) 데이터 디렉토리 : 데이터의 포인터(주소)를 저장하는 저장소
		2) 데이터 사전 : 부가정보(스키마, 스키마 간의 상호조건, 제약조건) 모두 저장된 저장소

메타 데이터 : 데이터베이스 자체에 관한 데이터( 한차원 높은 데이터라는 의미)


	10... 데이터 독립성
정의 : 스키마같은것이 변경되도 응용프로그램에게 영향이 적은것

	스키마 변경시 독립성 유지 방법 두가지
1) 외부/개념 사상 : 외부스키마가 변경되도 개념 스키마에 변경을 하지않거나 최소화하여 독립성유지(논리적데이터독립성)
2) 내부/개념 사상 : 내부 스키마가 변경되도 개념/내부 정보만 최소한으로 수정하여 독립성유지(물리적데이터독립성)
