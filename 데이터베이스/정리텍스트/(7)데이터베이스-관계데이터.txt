	5장 관계 데이터 모델
1. 관계 데이터 모델의 개념
	:개념적 구조를 논리적 구조로 표현하는 논리적 모델
	: 하나의 개체에 대한 데이터를 하나의 릴레이션에 저장
	: 릴레이션 : 테이블
2. 기본 용어 정리
	1) 릴레이션 : 객체, 2차원 테이블
	2) 속성 : 열,애트리뷰트
	3) 튜플 : 행
	4) 도메인 : 하나의 속성이 가질수있는 모든 집합
	5) 널 : 아무것도 가지지 않았을때 or 값이 없음
	6) 차주 : 속성의 갯수
	7) 카디널 리티 : 튜플의 전체갯수 : 행의 갯수, 세로로 몇개 있는지, 객체가 몇개 있는지
3. 릴레이션의 구성 = 스키마 + 인스턴스
	1) 릴레이션 스키마
		: 릴레이션의 논리적 구조(테이블의 논리적 구조) : 속성들행
	2) 릴레이션 인스턴스
		: 어느 한 시점에 릴레이션에 존재하는 튜플(객체)들의 집합 : 스키마행을 제외한 행들


	||데이터 베이스의 구성|| = 스키마 + 인스턴스
1. 데이터 베이스의 스키마 
	: 데이터 베이스 전체 구조, 데이터 베이스를 구성하는 릴레이션 스키마 모음
2. 데이터 베이스의 인스턴스
	: 릴레이션 인스턴스의 집합

	||릴레이션의 특징|| : 다중값 속성을 포함하기도 한다. =>(ex : 직업은 , 회사원, 학생이 될수있다.)

	
	||키|| : 투플들을 구별하는 속성또는 속성들의 집합

1. 키의 특성
	1) 유일성 : 하나의 릴레이션에서 모든 튜플은 서로 다른 키 값을 가져야함
	2) 최소성 : 꼭 필요한 속성들만 키로 구성됨
2. 키의 종류
	1) 슈퍼키 : 유일성을 만족하는 속성 or 속성들의 집합(여러가지 슈퍼키중 
	2) 후보키 : 유일성, 최소성을 만족하는 속성 or 속성들의 집합
	3) 기본키	: 특정지을수있는 특정한 키 
 	4) 외래키 : 다른 테이블에서 기본키가 되는  
 	5) 대체키 : 기본키가 되지 못한 후보키


	||관계 데이터 모델의 제약||
무결성 제약 조건
	1) 개체 무결성 제약조건
		: 기본키는 널값을 가질수 없다.
	2) 참조 무결성 제약조건
		: 외래키는 참조할수 없는 값을 가질수 없다.(없는 값을 참조못함)
