ㄱ. 관리시스템의 등장 배경

1...파일 시스템 : 파일 생성, 삭제, 수정, 검색 기능제공 소프트 웨어(데이터를 파일로 관리)

문제점
1) 중복저장(공간을 많이 차지함)
2)데이터 파일에 종속?적이다(파일 구조 변경시 프로그램 같이 변경)
3)공유, 보안, 회복 기능이 부족
4)응용 프로그램 개발 어렵다

ㄴ. 관리시스템의 정의

2...데이터 관리 시스템(DBMS)
파일 시스템의 문제를 해결하기 위해 제시된 소프트웨어
조직에 유리한 소프트웨어

주요 기능
1) 정의기능 : 정의, 수정 가능
2) 조작기능 : 삽입, 삭제, 수정, 검색 가능
3) 제어기능 : 데이터 정확, 안전 유지 가능

장점
중복x, 독립성, 동시 공유(중요), 보안, 무결성, 표준화,
회복빠름, 개발비용 감소

단점
고비용, 백업과 회복 방법 복잡

ㄹ. 관리시스템의 발전과정

1세대 : 네트워크 DBMS, 계층 DBMS
네트워크 : 데이터베이스를 그래프 형태로 구성
계층 : 데이터베이트 트리로 구성

2세대 : 관계 DBMS
데이터베이스를 테이블(표) 형태로 구성
ex)오라클, MS SQL서버, 엑세스, 인포믹스, 마이에스큐엘

3세대 : 객체지향 DBMS, 객체관계 DBMS
객체지향 : 객체를 이용해 데이터베이스를 구성
객체관계 : 객체 DBMS + 관계 DBMS

4세대
NoSQL DBMS
비정형 데이터에 적합, 뛰어남
안정성, 일관성, 유연성 좋다
데이터 분산, 저장, 처리하는 환경에 주로 사용

NewSQL DBMS : 관계 DBMS 장점 + NoSQL의 확장성
