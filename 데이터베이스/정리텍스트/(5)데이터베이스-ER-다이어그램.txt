2. 연산(오퍼레이션)

	4 . 개체-관계 모델(Entity-Relationship-diagram)

1. 개체(entity) = > 사각형으로 표시, 안에 이름 표기
	1) 사물 또는 사람
	2) 저장할 가치가 있는 중요데이터를 가지고 있는 사람
2. 속성(attribute) = > 타원으로 표현, 타원안에 이름 표기
	1) 개체,관계가 가지고 있는 고유한 특성
	2) 의미있는 데이터의 가장 작은 논리적 단위
3. 개체 타입(entity type) =>("고객","사람", 등등)
	1) 개체를 공유한 이름과 속성들로 정의한것
	2) 파일구조의 레코드 타입에 대응됨
4. 개체 인스턴스(entity instance)
	1) 속성이 실제 값을 가지면서 실체화된 개체
	2) 파일구조의 레코드 인스턴스에 대응됨
5. 개체 집합(entity set) =>(객체의 집합)
	1) 특정 개체 타입에 대한 개체 인스턴스들을 모아놓은 것
	

	5. 속성 구조

1. 단일값 속성과 다중값 속성
	1) 단일값 속성 : 값을 하나만 가질수있는 속성
	2) 다중값 속성 : 값을 여려개 가질수 있는 속성 => (이중 타원으로 표현)
2. 단순 속성과 복합 속성
	1) 단순 속성(심플 애트리뷰트) : 최소단위 속성
	2) 복합 속성(컴포짓 애트리뷰트) : 분해할수있는 속성
3 . 유도 속성 : 다른 속성들값에 의해 결정되는 속성(가격,할인율 => 판매가격)
4. 널 속성 : 널값이 허용되는 속성
5. 널값 : 존재하지 않는값 ( 공백이나  0과는 의미가 다름)
6. 키속성 : 각 인스턴스를 식별하는데 사용되는 속성(ex : ID같은 속성)=>밑줄로 표현
7. 관계 : 개체와 개체가 맺고있는 의미 있는 연관성 (ex : 고객은 책을 "구매"한다. ) 마름모로 표현
	1) 이항관계 : 개체 타입 2개가 맺는 관계
	2) 삼항 관계 : 3개가 맺는관계
	3) 순환 관계 : 1개가 자기 자신과 맺는 관계
	4) 관계의 유형 = 매핑 카디널리티 기준		= > 최솟값과 최댓값을 주기위해 실선에 (min,max)값을 양쪽에 적어준다.('*')은 값이 여러개 가능하다.
		1. 일대일 (1:1)
		2. 일대다 (1:n)
		3. 다대다 (n:m)			
8. 관계의 참여 특성
	1) 필수적 참여(전체 참여) : 모든 객체가 관계에 참여하는것(모든 고객은 책을 구매!) = > 이중선으로 표시
	2) 선택적 참여(부분 참여) : 일부만 참여(고객은 모두 책을 구매하지만 책에는 구매되지 않은 책이 있을수 있다.)
9. 관계의 종속성
	1) 약한 개체(weak entity) : 다른 개체의 존재 여부에 의존적인 개체 = > 이중사각형
	2) 강한 개체(strong entity) : 다른 개체의 존재 여부 결정하는 개체
	3) 약한개체와 강한개체는 = > 이중마름모로 표현

	
	6. 논리적 데이터 모델
1. 논리적 데이터 모델 : 사용자가 생각하는 데이터베이스에 저장할 형태의 구조
2. 관계 데이터 모델 : 데이터베이스 구조가 2차원 테이블 형태
3. 계층 데이터 모델 : 트리형태(상속관계) 1:n만 허용
4. 네트워크 데이터 모델 : 그래프형태 다대다 허용
	

	7. 물리적 데이터 모델링
실제 데이터 베이스에 저장하기 위해 구조를 정의하고 구현하는 과정
