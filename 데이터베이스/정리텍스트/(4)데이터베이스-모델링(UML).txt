	3 . UML구성요소
1) 사물 :
	ㄱ. 정적사물 : 개념적,물리적 요소
		1. 클래스 : 객체를 기술한것(직사각형 표시)
		2. 인터페이스 : 클래스,컴포넌트의 서비스를 명세화하는 연산을 모아놓은것
		3. 통신 : 서로다른 요소와 역할이 모여 교류를 정의(실선 사각형 표시)
		4. 컴포넌트 : 전체시스템을 구성하는 단위(탭이달린 사각형 표시)
		5. 패키지 : 요소를 그룹으로 묶음(개념적)(탭이 달린 폴더 형태 표시)(폴더라 생각하면 됌)
		6. 노드 : 실행할때 존재하는 물리적 요소(육면체 표시)
	ㄴ. 동적 사물 : 동적인 부분을 동사로 표시
		1) 교류 : 객체간 메시지로 구성된 동작(화살표표시)
		2) 유스케이스(Use case) : 수행하는 활동들을 순차적 기술(실선 타워 표시)
		3) 상태머신 : 외부 이벤트에 대한 객체의 상태,상태변화 순서 기술(모서리 둥근 직사각형 표현)
	ㄷ. 주해사물
		1)노트 : 첨부되는 주석,제약을 기술(모서리가 접힌 사각형)

2) 관계
	ㄱ. 의존 관계 : 두 사물간 의미적 관계, 의존되는 사물이 변하는 의존하는 사물이 변한다. (의존하는 사물을 향해 점선 화살표)
	ㄴ. 연관 관계 : 객체 사이,지속적으로 연결 유지 되는 관계(실선 표시, 이름 기술(의미설명과 방향삼각형 표시),양쪽 객체에 역할을 적는다.)
		1. 다중성 : 몇개가 연결되어있는지 숫자로 객체역할을 적는것(ex:1..* = 하나 이상)
		2 .집합 연관 : 트리 구조, 부모가 자식을 소유(상속의 반대)
		3. 일반화 관계 : 트리구조, 자식이 부모를 소유(상속)
		4. 실체화 관계 : 한객체가 다른객체에게 연산을 수행하도록 지정하는 관계(ex : 텔레비젼 - 리모컨)
3) 다이어그램
	1. 정적 다이어그램
		ㄱ. 클래스 다이어그램 : 클래스,인터페이스,통신과 함께 이들의 관계를 나타냄
		ㄴ. 컴포넌트 다이어그램 : 컴포넌트 사이의 구성과 의존 표현
		ㄷ. 배치 다이어그램 : 실행시 처리하는 노드, 그 노드에 있는 컴포넌트들의 구성 표현
		ㄹ. 패키지 다이어그램 : 여러 모델 요소 그룹화, 패키지 사이를 관계로 표현
	2. 동적 다이어그램
		ㄱ. 유스케이스 다이어그램 : 유스케이스와 액터관계 구조적으로 표현
		ㄴ . 순차 다이어그램과 통신 다이어그램 : 교류 다이어그램의 한종류
		ㄷ . 활동 다이어그램 : 시스템 내부에 있는 활동의 흐름 표현
		ㄹ . 상태 다이어그램 : 시스템의 동적뷰 나타냄
